Pe risolvere ssrf basics level 4 è presente una discrepranza di parsing.
PHP e curl non concordano su chi è l'host quando ci sono due @ nell'url(Inventato da Orange Tsai)
Ho pensato di creare un url con due @ in modo tale che parse_url vede solo l'ultima, mentre curl vede solo la prima
L'url è: http://foo@127.0.0.1:80@8.8.8.8/get_flag.php
Dove parse_url vede come credenziali:
	user = foo@127.0.0.1:80
	host = 8.8.8.8
	path ) /get_flag.php
Invece curl vede solo la prima @ come separatore:
	user = foo
	host = 127.0.0.1:80  <- si connette qui
	path = /get_flag.php
Quindi il flusso completo è:
1) filter_var -> URL valido
2)parse_url -> host = 8.8.8.8 -> gethostbyname -> IP pubblico
3)cidr_match -> 8.8.8.8 non è 127.x.x.x -> passa
4) curl -> si connette a 127.0.0.1/get_flag.php -> FLAG
